\subsection{{\href{https://github.com/JustUzair/open-source-local-auditor}{SentinelAI | Local-First AI Security Auditor}}}
\begin{zitemize}
  \item Ingests \textbf{550+ open source security reports} into a local \textbf{HNSWlib} vector store (\textbf{~19,500 chunks}), K-Means clustered ($K=10$) into vulnerability classes; four-phase pipeline — deterministic pre-scanner narrows attack surface at zero LLM cost, \textit{Protocol Map} cartographer extracts call graphs and entry points, iterative passes propagate suspicions across rounds, Supervisor deduplicates and severity-ranks the final report.
  \item Benchmarked against a real private audit: local \textbf{qwen3.5:27b} matched \textbf{6/8 professional findings} and surfaced \textbf{8+ additional valid vulnerabilities} the paid audit missed, runs fully local, zero API cost.
  \item \textbf{Tech}: TypeScript, Ollama, HNSWlib, K-Means, RAG, LangChain, Solodit
\end{zitemize}

\subsection{{OpenClaw | Agentic Job Search Engine}}
\begin{zitemize}
  \item A tool I built and use daily; aggregates jobs across \textbf{HN Algolia}, \textbf{Greenhouse}, \textbf{Lever}, and \textbf{Ashby} ATS APIs via async \textbf{aiohttp} fan-out; auto-discovers company slugs from HN ``Who is Hiring'' threads, scheduled daily with title regex pre-filtering before any LLM call to eliminate irrelevant roles cheaply.
  \item \textbf{LLM resume tailoring}: reads \textbf{LaTeX source} directly, rewrites \texttt{experience}, \texttt{projects}, and \texttt{summary} against the target JD, auto-repairs malformed JSON, and compiles a \textbf{production PDF via \texttt{latexmk}}, variants persisted in SQLite with per-variant refinement loop.
  \item \textbf{Cold outreach engine}: scrapes LinkedIn profiles, infers emails from name patterns, generates personalized email templates via local Ollama LLM, dispatches bulk SMTP with rate limiting, name to sent email, zero manual steps.
  \item \textbf{Tech}: Python, FastAPI, React, SQLite, Docker, Ollama
\end{zitemize}

\subsection{{\href{https://github.com/JustUzair/codecrafters-redis-go}{GoRedis}}}
\begin{zitemize}
  \item \textbf{Wire-compatible Redis server} in Go with \textbf{zero external dependencies}, with only stdlib; accepts native \textbf{\texttt{redis-cli}} with no client changes, custom \textbf{RESP parser} over raw TCP, \textbf{generic \texttt{Storage[T~any]}} with \textbf{RWMutex}-guarded reads/writes and ms/s TTL granularity.
  \item Full list semantics with correct negative-index clamping; \textbf{BLPOP} via channel-based notifier, callers park on a per-key \texttt{chan~struct\{\}} bell that \texttt{LPUSH} rings, no polling; \textbf{Streams} (\texttt{XADD}/\texttt{XRANGE}) with auto-ID, auto-sequence, and monotonicity enforcement; deployed as a \textbf{stateful K8s workload} with local persistence snapshot for cross-replica state sharing.
  \item \textbf{Tech}: Go, TCP/RESP, generics, channels, mutexes, Docker, docker-compose, Kubernetes (K8s),  Redis
\end{zitemize}
